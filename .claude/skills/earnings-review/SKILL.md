# Earnings Review Skill

## When to use
用户希望得到一份**结构稳定**的财报点评，而不是天马行空的描述。

## Workflow

1. 调用 `GET /api/earnings/{code}` 取财报快评原始数据。
2. 调用 `GET /api/earnings/{code}/history?limit=8` 取最近 8 期摘要。
3. 按下面固定结构组织输出。**不要改变章节顺序**。

## Output template

```
# {名称}（{代码}）{报告期} 财报点评（draft）

## 1. 一句话结论
（≤30 字，定性，不带涨跌建议）

## 2. 关键数据
| 指标 | 本期 | 同比 | 环比 |
| --- | --- | --- | --- |
| 营业收入 | … | … | … |
| 归母净利 | … | … | … |
| 扣非EPS | … | — | — |
| 加权ROE | … | — | — |

## 3. 同比/环比解读（3 条以内）
- …

## 4. 利润质量（必写）
- 扣非EPS / 基本EPS = X%，{描述}
- 经营性现金流 vs 净利匹配度：（如数据缺失则注明 "数据缺失，建议查阅现金流量表"）

## 5. 多期趋势（5 行内）
- 营收增速：… 趋势
- 净利增速：… 趋势

## 6. 风险点（系统识别 + 人工补充建议）
- 系统识别：{red_flags}
- 建议补充核查：（列 2-3 项）

## 7. 复核 checklist
- [ ] 财务数据与年报/公告一致
- [ ] 非经常性损益拆解
- [ ] 行业景气度对照
- [ ] 是否有会计政策变更

---
免责声明：本点评为 AI 草稿（draft），不构成投资建议，请由具备资格的专业人士复核。
```

## Hard rules
- 标题必须含 `(draft)`
- 第 7 节"复核 checklist" 必写
- 永远不写"买入/卖出/目标价"
- 缺失数据用 `—`，禁止编造
